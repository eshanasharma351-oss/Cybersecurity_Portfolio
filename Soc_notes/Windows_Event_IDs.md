# Windows Event IDs

## Overview
Windows Event IDs are logs generated by the operating system to record important activity on a system. In SOC work, they help analysts detect suspicious behavior, investigate incidents, and understand what happened on an endpoint .

## What Windows Event IDs Are
Each event ID represents a specific type of action, such as a login, account change, process creation, service installation, or audit change. These logs become more useful when they are correlated with other telemetry like Sysmon, PowerShell logs, and network activity .

## Why They Matter in SOC
Windows Event IDs are important because many attacks leave traces in authentication, persistence, privilege escalation, and lateral movement events. A good analyst can use them to spot brute force attempts, unauthorized account changes, malware execution, and log tampering .

## Authentication Events
Authentication events help track who logged in, how they logged in, and whether the attempt succeeded or failed.

Common Event IDs:
- 4624 = Successful logon.
- 4625 = Failed logon.
- 4648 = Logon with explicit credentials.
- 4672 = Special privileges assigned to a logon.
- 4776 = NTLM authentication attempt.

## Account Management Events
Account management events show changes to users and groups. These are important because attackers often create accounts or add themselves to privileged groups to maintain access.

Common Event IDs:
- 4720 = User account created.
- 4722 = User account enabled.
- 4725 = User account disabled.
- 4732 = Member added to a security-enabled local group.
- 4740 = Account locked out .

## Process Creation Events
Process creation events help reveal what ran on the machine and whether anything unusual was launched.

Common Event IDs:
- 4688 = Process created.
- 4104 = PowerShell script block logging.
- 7045 = New service installed.
- 4697 = Service installed .

## Persistence-Related Events
Attackers often use Windows features to survive reboots and stay on the system. Event IDs for scheduled tasks, services, and registry changes are useful for spotting that behavior.

Common Event IDs:
- 4698 = Scheduled task created.
- 4697 = Service installed.
- 4657 = Registry value modified, when auditing is enabled.
- 7045 = New service installed .

## Privilege Escalation Events
Privilege escalation events show when an account gains higher-level access than normal. These events can reveal misuse of admin rights or compromised privileged accounts.

Common Event IDs:
- 4672 = Special privileges assigned to a logon.
- 4728 = Member added to a security-enabled global group.
- 4732 = Member added to a security-enabled local group.
- 4756 = Member added to a security-enabled universal group .

## Lateral Movement Events
Lateral movement events help identify when an attacker tries to move from one system to another. These are especially useful when paired with logon and share-access activity.

Common Event IDs:
- 4648 = Logon attempt using explicit credentials.
- 5140 = Network share accessed.
- 5145 = Detailed network share access.
- 4768 = Kerberos TGT request.
- 4769 = Kerberos service ticket request .

## Audit Tampering Events
Attackers may try to hide by clearing logs or changing audit settings. These events are strong warning signs and should be reviewed carefully.

Common Event IDs:
- 1102 = Security log cleared.
- 4719 = Audit policy changed.
- 4902 = Audit policies updated .

## Useful Log Sources
Windows Security logs are the main source for Event IDs, but they are not the only source worth collecting. Sysmon, PowerShell Operational logs, Task Scheduler logs, and Defender logs can all provide extra detail for investigations .

## Detection Tips
A single event does not always mean compromise, so context matters. Look for unusual timing, new source hosts, rare accounts, strange parent-child processes, or repeated activity across several related events .

## Investigation Example
If you see Event ID 4625 followed by 4624, then 4672, and later 4688 launching PowerShell, that may point to suspicious access followed by privileged execution. If you also see 4698 or 7045, persistence may be involved .

## Key Takeaway
Windows Event IDs are one of the most important sources of evidence for SOC analysts. The best investigations come from combining authentication, process, persistence, and audit events into one timeline .
